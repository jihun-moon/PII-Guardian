종합 OCR (이미지) 유출 테스트 (v2)

이 페이지는 PII-Guardian 봇이 CLOVA OCR을 통해 이미지 속 텍스트 PII를 잘 탐지하는지 테스트합니다.

(참고: 이 이미지들은 실제 파일이 아닌, 텍스트가 포함된 placeholder URL입니다.)

(시나리오 1) 명백한 '유출' (가짜 내부 스크린샷)

'신입' 봇이 이미지 속 텍스트(010-1234-5678, backup-dev@...)를 읽어야 합니다.
'전문가' 봇(LLM)은 이 문맥을 보고 '유출(Leak)'로 판단해야 합니다.

(시나리오 2) 명백한 '공개' (가짜 명함/고객센터)

봇이 이미지 속 텍스트(1588-5050, help@dgb.co.kr)를 읽어야 합니다.
'전문가' 봇(LLM)은 '고객센터', '대표전화' 등의 문맥을 보고 '공개(Public)'로 판단해야 합니다.

(시나리오 3) AI-NER 탐지 (가짜 포스터)

봇이 이미지 속 텍스트('홍길동', 'DGB대구은행')를 읽어야 합니다.
'신입' 봇의 AI-NER가 '홍길동(PS)'과 'DGB대구은행(OG)'을 탐지해야 합니다.

(시나리오 4) PII가 없는 이미지 (안전)

봇이 이 이미지를 스캔(OCR)하지만, PII가 없으므로 아무것도 탐지하지 않아야 합니다. (False Positive 테스트)

(시나리오 5) 금융 정보 '유출' (가짜 계좌 스크린샷)

봇이 DGB 계좌번호, 카드번호를 읽어내야 합니다. '유출(Leak)'로 판단되어야 합니다.

(시나리오 6) 최고 등급 민감 정보 '유출' (신분증)

봇이 주민번호, 이름 등을 읽어내야 합니다. '유출(Leak)'로 판단되어야 합니다.

(시나리오 7) 내부 기술 정보 '유출' (코드 스크린샷)

봇이 이미지 속 API 키, 내부 IP 등을 읽어내야 합니다. '유출(Leak)'로 판단되어야 합니다.
