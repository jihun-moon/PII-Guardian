(중략) ## 🔧 설정 방법 (v3.0 기준) 1. `config.ini` 파일을 생성합니다. 2. 아래 코드를 복사하여 붙여넣으세요. ```ini [Database] Host = localhost Port = 3306 ; (오탐: 포트번호) User = root ; 임시 비밀번호: MyS3cret_DB_Pass! (유출) [API] ; 네이버 API (테스트용) ClientID = _aBcDeFg123456XyZ (유출) ClientSecret = G7h8J9k0L1 (유출) ```` ## 🐛 버그 리포트 - 이슈 트래커: `https://github.com/my-project/issues` - 긴급 연락: \010-8888-9999\ (프로젝트 관리자) (유출) - 이메일 문의: [support@my-project.com](mailto:support@my-project.com) (공개) - 담당자 개인 이메일: [dev.choi@gmail.com](mailto:\dev.choi@gmail.com\) (유출) ```` --- ### 시나리오 2: Raw JSON (`api_response.json`) 서버에서 클라이언트로 전송된 API 응답 데이터입니다. 민감한 정보가 그대로 노출된 상황입니다. ```json { "status": "success", "request_id": "req-9a8b7c6d-5e4f-3a2b-1c0d", // (오탐: UUID) "data": [ { "userId": 101, "username": "User1", "contact": { "email": "user1@gmail.com", "phone": "010-1111-2222" }, "shipping_address": { "street": "서울시 강남구 테헤란로 123", "detail": "OO빌딩 501호", "zipcode": "06123" // (오탐: 공개된 우편번호) }, "last_login_ip": "211.123.45.67" // (유출: IP 주소) }, { "userId": 102, "username": "User2", "contact": { "email": "user2@naver.com", "phone": "010-3333-4444" }, "shipping_address": { "street": "대구광역시 중구 동성로 15", "detail": "XX아파트 102동 304호", "zipcode": "41911" // (오탐: 공개된 우편번호) }, "last_login_ip": "192.168.0.1" // (유출: 내부 IP도 포함) } ], "version": "v2.1.5" // (오탐: 버전 정보) } ```` ----- ### 시나리오 3: 텍스트 로그 (`chat_log.txt`) 고객센터 채팅 로그입니다. 사용자가 문제를 해결하기 위해 개인정보를 입력하는 상황입니다.
