# 🤖 Pii-Guardian: AI MLOps 포트폴리오

**Python, MLOps(CI/CD, CT), NCP(Naver Cloud Platform)를 활용하여 구축한 AI 기반 개인정보 유출 탐지 봇 프로젝트입니다.**

이 시스템은 3개의 AI 봇이 유기적으로 데이터를 수집, 판독, 재학습하며 스스로 성능을 고도화하는 'AI 팩토리' (MLOps 파이프라인)로 설계되었습니다.

---

## 🏛️ 1. MLOps 아키텍처

이 AI 팩토리는 3개의 봇(Bot)이 파이프라인으로 연결되어 24시간 자동으로 실행됩니다.

* **1. '신입' 봇 (`crawler.py`)**
    * **역할:** 1차 탐지 (데이터 수집)
    * **기능:** 정해진 주기를 돌며 웹사이트를 크롤링합니다. 자체 AI 모델(`my-ner-model`)과 정규식(Regex)을 사용해 '의심'되는 개인정보를 수집하여 `detected_leaks.csv`에 저장합니다.

* **2. '전문가' 봇 (`autolabeler.py`)**
    * **역할:** 2차 판독 (데이터 레이블링)
    * **기능:** '신입' 봇이 수집한 '의심' 목록을 확인합니다. `llm_helper.py`를 통해 Naver HyperCLOVA X API에게 "이것이 진짜 유출인지, 오탐인지"를 판단시켜, '정답'을 `feedback_data.csv`에 저장합니다.

* **3. '학습기' 봇 (`train.py`)**
    * **역할:** 자동 재학습 (지속적 학습, CT)
    * **기능:** '전문가' 봇이 생성한 '정답' 데이터를 학습하여 '신입' 봇의 뇌(`my-ner-model`)를 **자동으로 업그레이드(Fine-tuning)**합니다.

* **4. 중앙 관제소 (`dashboard.py`)**
    * **역할:** 모니터링
    * **기능:** Streamlit으로 구축된 웹 대시보드. 3개 봇의 실시간 현황, 스케줄, 로그 및 탐지 데이터를 한눈에 모니터링합니다.

## 🏦 2. '가상 은행' 테스트 서버 (`test_site`)

이 리포지토리에는 PII 봇의 성능을 테스트하기 위한 **'가상 은행' (문제 출제) 서버**가 포함되어 있습니다. (`test_site/app.py`)

* **기능:** `Faker` 라이브러리를 사용해 실시간으로 고난이도 PII(주민번호, 카드번호, Base64, 난독화 패턴 등)가 포함된 HTML 페이지를 생성합니다.
* **활용:** 이 프로젝트(`crawler.py`)의 테스트 타겟으로 사용됩니다.
* **(별도 활용 가능):** `app.py` 서버를 실행하면, **n8n, AutoGPT 등 다른 AI 에이전트 프로젝트**의 데모를 위한 '문제 출제 서버'로 독립적으로 활용할 수 있습니다.

## ⚙️ 3. 로컬 데모 실행 방법

(NCP 서버가 아닌 로컬 환경에서 수동으로 데모를 실행하는 방법입니다.)

1.  **가상 환경 활성화:**
    ```bash
    source venv/bin/activate
    ```

2.  **(터미널 1) '가상 은행' 서버 실행:**
    ```bash
    cd test_site
    python3 app.py
    ```
    * `http://127.0.0.1:5000/` (또는 `http://<내부IP>:5000/`)에서 서버가 실행됩니다.

3.  **(터미널 2) '중앙 관제소' 실행:**
    ```bash
    streamlit run dashboard.py
    ```
    * `http://localhost:8501`에서 대시보드가 열립니다.

4.  **(터미널 3) MLOps 파이프라인 수동 실행:**
    * **1차 탐지:** `python3 crawler.py`
    * **2차 판독:** `python3 autolabeler.py`
    * **재학습:** `python3 train.py`

    (실행 후, 대시보드의 '현황판 새로고침' 버튼을 누르면 `처리 대기` 건수 등이 변경됩니다.)
